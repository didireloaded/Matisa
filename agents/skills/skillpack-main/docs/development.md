# Development Guide

```text
skill-pack/
├── src/                          # TypeScript source
│   ├── cli.ts                    # CLI entry point
│   ├── commands/

│       ├── server.ts
│       ├── agent.ts
│       ├── config.ts
│       ├── lifecycle.ts
│       └── adapters/
│           ├── types.ts
│           ├── web.ts
│           ├── telegram.ts
│           ├── slack.ts

├── skillpack.json                # Example pack config (generated by create)

```

## npm Scripts

These scripts are defined in `package.json` for day-to-day development.

| Script | Command | Description |
| --- | --- | --- |
| `npm run build` | `tsup` | Compile `src/` → `dist/` (CLI + runtime in one pass) |
| `npm run dev` | `tsup --watch` | Watch mode — recompile on file changes |
| `npm run check` | `tsc --noEmit` | Type-check without emitting output |
| `npm run format` | `prettier --write .` | Format all source files |
| `npm run create` | `tsup && node dist/cli.js create` | Build then run `create` |
| `npm run run` | `tsup && node dist/cli.js run` | Build then run `run` |
| `npm run zip` | `tsup && node dist/cli.js zip` | Build then run `zip` |

The `create`, `run`, and `zip` scripts rebuild first so you always test against the latest compiled output. Pass extra arguments after `--`:

```bash
npm run create -- output
npm run create -- output --config ./fixtures/skillpack.json
npm run run -- output
npm run run -- . --port 3000
```

## CLI Reference

After building, the CLI is available as `node dist/cli.js` (or `npx @cremini/skillpack` after install).

### `skillpack create [directory]`

Interactively create a new pack.

```bash
skillpack create
skillpack create output
skillpack create output --config ./fixtures/skillpack.json
skillpack create output --config https://example.com/skillpack.json
```

| Option | Description |
| --- | --- |
| `[directory]` | Target directory (created if it does not exist). Defaults to `cwd`. |
| `--config <path-or-url>` | Skip the interactive wizard and initialize from a local file or remote URL instead. |

**Interactive mode** (no `--config`):

1. Prompts for `App name` and `Description`.
2. Collects one or more skill sources and names in a loop.
   - Source formats: `owner/repo`, full GitHub URL, or a local `./skills/` path.
   - Inline skill names are supported: `owner/repo --skill skill-name`.
3. Collects one or more prompts in a loop.
4. Asks whether to package a zip immediately.
5. Saves `skillpack.json`, copies `start.sh` / `start.bat`, installs skills.

**`--config` mode** (replaces the old `init` command):

1. Fetches the config from a local path or HTTPS URL.
2. Validates the config structure.
3. Saves `skillpack.json`, installs all declared skills, syncs descriptions, copies start scripts.

### `skillpack run [directory]`

Start the runtime server.

```bash
skillpack run
skillpack run output
skillpack run output --port 3000
skillpack run output --host 0.0.0.0
```

| Option | Description |
| --- | --- |
| `[directory]` | Pack root directory. Defaults to `cwd`. Created if it does not exist. |
| `--port <port>` | Port to listen on. Sets `PORT` env var. Default: `26313`. |
| `--host <host>` | Address to bind to. Sets `HOST` env var. Default: `127.0.0.1`. |

**Behavior:**

1. If `skillpack.json` is missing in the target directory, prompts for `name` and `description`, generates a default config, and copies start scripts.
2. Loads the config and compares declared remote skills against `skills/`.
3. Auto-installs any missing remote skills (local `./skills/` sources are skipped).
4. Starts the runtime server and opens the browser.

### `skillpack zip`

Package the current pack as a distributable zip file.

```bash
skillpack zip
```

No options. Always runs in `cwd`.

**What gets packaged:**

| Path in zip | Source |
| --- | --- |
| `<name>/skillpack.json` | `skillpack.json` in cwd |
| `<name>/job.json` | optional `job.json` in cwd |
| `<name>/AGENTS.md` | optional `AGENTS.md` in cwd |
| `<name>/SOUL.md` | optional `SOUL.md` in cwd |
| `<name>/skills/` | `skills/` directory |
| `<name>/start.sh` | `start.sh` in cwd (execute bit preserved) |
| `<name>/start.bat` | `start.bat` in cwd |

**Behavior:**

1. Reinstalls all configured skills.
2. Scans `skills/` for `SKILL.md` frontmatter and syncs descriptions back into `skillpack.json`.
3. Includes `job.json` when the file exists at the pack root.
4. Produces `<config.name>.zip` in `cwd`.

The output zip is intentionally minimal — no server or web UI is bundled. End users run `start.sh` which invokes `npx @cremini/skillpack run .` to fetch the runtime from npm.

## Environment Variables

These variables are read by the runtime at startup and can override runtime fields in `data/config.json`. Scheduled jobs are loaded separately from pack-root `job.json`.

| Variable | Description | Default |
| --- | --- | --- |
| `PACK_ROOT` | Pack root directory | argument passed to `startServer` |
| `HOST` | Listen address | `127.0.0.1` |
| `PORT` | Listen port | `26313` |
| `SKILLPACK_API_KEY` | Highest-priority runtime API key override | — |
| `SKILLPACK_PROVIDER` | Highest-priority runtime provider override | — |
| `SKILLPACK_BASE_URL` | Highest-priority runtime base URL override | — |
| `SKILLPACK_API_PROTOCOL` | Highest-priority runtime API protocol override | — |
| `SKILLPACK_REASONING` | Highest-priority runtime reasoning flag override | — |
| `OPENAI_API_KEY` | OpenAI API key fallback | — |
| `ANTHROPIC_API_KEY` | Anthropic API key fallback | — |

Priority order for PackAgent runtime auth/config is:

1. `SKILLPACK_*`
2. `data/config.json`
3. Provider fallback env vars such as `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`

In embedded hosts such as Frevana, prefer injecting `SKILLPACK_*` at process start instead of writing those values into `data/config.json`.

Example:

```bash
PORT=4000 OPENAI_API_KEY=sk-... node dist/cli.js run output
```

## Common Workflows

### Start developing a new pack from scratch

```bash
npm install
npm run create -- my-pack
cd my-pack
npm run run
```

### Initialize from an existing remote config

```bash
npm run create -- my-pack --config https://example.com/skillpack.json
npm run run -- my-pack
```

### Rebuild and test the runtime quickly

```bash
npm run run -- output
```

### Package for distribution

```bash
cd my-pack
skillpack zip
# → my-pack.zip
```
