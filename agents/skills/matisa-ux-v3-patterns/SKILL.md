# Matisa UX Overhaul V3 Patterns

Matisa is **people-first**, not content-first. Every screen should create interaction, not passive consumption. Adhere to these principles when building new features:

## 1. Action-Oriented Empty States
Never show a generic "No Data" message.
Instead, use an **Invite Action**.
- If there are no Rooms: "Create a Room"
- If there are no Events: "Create an Event"
- If there are no Notes: "Ask Your First Question"

## 2. Quick Action Bar
Always provide a maximum one-tap entry to core creation features: Record Voice, Create Note, Create Room, Create Event, Ask Question, Post Opportunity.

## 3. Social Proof Visibility
Whenever recommending or listing content (Rooms, Events, People), surface social context.
- **Always show:** Mutual Connections, Shared Events, Shared Rooms, Shared Interests, Shared Creators.
- Make strangers feel less like strangers.

## 4. Notes are Conversations
Notes should not be passive text blocks.
Every Note must display actionable buttons: **Reply, Voice Reply, Message User, Join Discussion**.

## 5. Voice is Frictionless
Recording, playback, and publishing must take exactly **one tap**. Ensure Voice forms a core component across Notes, Stories, and Profiles.
