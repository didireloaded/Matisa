# MATISA UX OVERHAUL V3

## CORE UX PHILOSOPHY
Matisa is not content-first. Matisa is people-first.
Users should always know:
* Who is here
* What is happening
* Where they can join
* Who they can meet
* What opportunities exist
Every screen should create interaction. Not passive consumption.

---

# HOME UX
Current social apps: Open app → Scroll
Matisa: Open app → Join
Home should answer: What's happening right now? Who is active? Which rooms are live? Which events are coming up? Who needs help? Who wants to collaborate?
Home Structure:
Stories
Active Voice Rooms
Trending Notes
People To Meet
Upcoming Events
Creator Opportunities
Trending Karaoke

---

# QUICK ACTION BAR
Always visible.
Actions: Record Voice, Create Note, Create Room, Create Event, Ask Question, Post Opportunity
Maximum one tap.

---

# NOTES UX
Current notes are too passive. Turn notes into conversations.
Examples: Need a videographer Saturday. Anyone going to this event? Looking for a singer. Need help editing a film.
Every note should have: Reply, Voice Reply, Message User, Join Discussion

---

# VOICE UX
Voice should be easier than typing.
Recording should take: One tap.
Playback should take: One tap.
Publishing should take: One tap.
Voice Features: Voice Notes, Voice Stories, Voice Replies, Voice Polls, Voice Questions, Voice Intros, Voice Requests

---

# PROFILE UX
Profile should become a social resume.
Immediately show: Photo, Voice Intro, Bio, Location, Availability, Current Status, Upcoming Events, Recent Rooms, Recent Notes
Availability Status: Available, Busy, Open To Collaborate, Looking For Work, Hiring

---

# CREATOR UX
Creators need immediate value.
Show: Recent Work, Availability, Top Skills, Open Projects, Upcoming Events, Voice Intro

---

# DISCOVERY UX
People first. Not content first.
Discovery sections: People, Creators, Events, Rooms, Voice, Opportunities, Nearby
Every recommendation should explain itself.
Examples: You both like photography. 3 mutual connections. Attending the same event. Joined similar rooms.

---

# ROOM UX
Room cards should immediately answer: What is this room? Who is inside? Should I join?
Show: Topic, Host, Participants, Friends Inside, Live Status, Join Button

---

# KARAOKE UX
Karaoke should feel like a live show. Not a chat room.
Before joining: Current Singer, Current Song, Audience Count, Queue Length, Top Performer
Audience Actions: Cheer, React, Vote, Follow Singer, Request Song, Join Queue
After Performance: Performance Score, Audience Rating, New Followers, New Friends, Achievements

---

# EVENTS UX
Events should feel social before they happen.
Show: Friends Attending, Creators Attending, Mutual Connections, Room Discussions, Event Voice Room, Questions
Users should feel involved before arriving.

---

# OPPORTUNITY UX
New Major Feature: Opportunity Board
Types: Jobs, Gigs, Collaborations, Casting Calls, Talent Requests, Crew Requests
Opportunity cards should include: Role, Budget, Location, Date, Creator, Apply Button, Message Button, Voice Intro

---

# MESSAGING UX
Make conversations feel alive.
Support: Text, Voice, Images, Video, Shared Notes, Shared Events, Shared Rooms
Conversation header should show: Mutual Interests, Mutual Events, Mutual Friends

---

# SOCIAL PROOF UX
Show: Mutual Connections, Shared Events, Shared Rooms, Shared Interests, Shared Creators
This makes strangers feel less like strangers.

---

# EMPTY STATES
Never show: No Data
Instead: Invite Action
Examples: No Rooms -> Create Room, No Events -> Create Event, No Notes -> Ask Your First Question

---

# RETENTION UX
Every session should produce: A new person, a new room, a new opportunity, a new event, a new conversation, a new voice interaction.
Users should never leave thinking: Nothing happened.

---

# FINAL UX GOAL
When users open Matisa they should immediately feel:
People are here. Conversations are happening. Opportunities exist. Events are happening. Rooms are active. I can participate right now.
That feeling should drive every UX decision across the platform.
